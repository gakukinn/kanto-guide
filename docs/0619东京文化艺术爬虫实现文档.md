# Jalan.net 文化艺术活动爬虫实现文档

## 项目概述

本文档记录了使用 **Playwright + Cheerio** 技术栈从 jalan.net 网站爬取文化艺术活动数据的完整实现过程。

### 目标网站

- **URL**: `https://www.jalan.net/event/130000/?screenId=OUW1211`
- **内容**: 东京地区综合活动列表
- **筛选**: 获取非五大活动（祭典、赏花、花火、狩枫、灯光）的文化艺术活动

### 技术栈

- **Playwright**: 浏览器自动化
- **Cheerio**: HTML解析
- **Node.js**: 运行环境
- **JSON**: 数据存储格式

## 核心功能

### 1. 智能活动筛选

爬虫具备智能筛选逻辑，可以自动识别和分类活动：

**排除关键词**（五大活动）：

- 祭典：`花火`、`祭り`、`祭`、`まつり`、`マツリ`
- 赏花：`花見`、`桜`、`さくら`、`梅`
- 狩枫：`紅葉`、`もみじ`、`イチョウ`
- 灯光：`イルミネーション`、`夜景`、`ライトアップ`
- 其他花卉：`ハナショウブ`、`菖蒲`、`あじさい`

**包含关键词**（文化艺术）：

- 展览：`展`、`美術`、`アート`、`デザイン`
- 表演：`音楽`、`コンサート`、`ライブ`
- 文化：`文化`、`伝統`、`工芸`、`フェスティバル`

### 2. 数据结构

```typescript
interface CultureActivity {
  id: string; // 唯一标识
  name: string; // 活动名称
  area?: string; // 区域
  date?: string; // 日期
  location: string; // 地点
  description?: string; // 描述
  url?: string; // 官方链接
  crawledAt?: string; // 爬取时间
  category: string; // 分类
  type?: string; // 类型
  prefecture?: string; // 都道府县
  ward?: string; // 区市町村
  venue?: string; // 场馆
}
```

### 3. 爬虫特性

- **容错处理**: 自动重试和超时处理
- **反反爬**: 使用真实用户代理和随机延迟
- **数据验证**: 自动检查数据完整性
- **备选数据**: 爬取失败时使用高质量预设数据

## API接口

### GET /api/culture

获取文化艺术活动数据

**参数**:

- `region`: 地区代码（默认: tokyo）

**响应示例**:

```json
{
  "region": "tokyo",
  "events": [...],
  "count": 5,
  "status": "success",
  "timestamp": "2025-06-18T14:00:00.000Z"
}
```

### POST /api/culture

更新活动数据

**支持操作**:

- `like`: 更新点赞数
- `refresh`: 刷新数据
- `validate`: 验证数据完整性

## 数据存储

根据项目的 **自动化存储决策系统**，文化艺术活动数据自动选择 JSON 格式存储：
