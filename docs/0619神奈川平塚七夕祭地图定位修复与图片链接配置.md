## 错误识别与修正方法

### 关键错误案例

**错误**：在FeaturedActivities组件中设置错误的活动日期信息
**具体问题**：

- 成田祇园祭：错误显示"7月第二个周末"，实际为"7月4-6日"
- 多个活动使用模糊的"第X个周末"表述

### 修正方法

1. **数据源验证**：

   - 必须从数据库或官方文件中获取准确日期
   - 禁止使用推测性或模糊的日期表述
   - 每次修改前先查询现有数据源

2. **验证流程**：

   ```typescript
   // 修改前必须执行的验证步骤
   1. 搜索数据库中的准确信息
   2. 对比多个数据源确认一致性
   3. 使用具体日期而非相对表述
   ```

3. **错误防范机制**：
   - 建立数据准确性检查工具
   - 要求所有日期信息有明确来源标注
   - 定期验证热门推荐中的活动信息

## 技术规范总结

### 地图坐标管理规范

- 活动地点应定位到最相关的交通枢纽或主要会场
- 坐标精度保持到小数点后7位
- 同时更新页面嵌入URL和数据文件链接

### 图片资源管理规范

- 使用实际存在的图片文件路径
- 支持多种图片格式以优化加载性能
- 保持SEO元数据与实际图片的一致性

### 数据准确性管理规范

- **绝对禁止**：使用未经验证的日期信息
- **必须执行**：从可靠数据源获取所有关键信息
- **错误处理**：发现错误时立即暂停并重新验证

## 日后AI工作指导

### 关键注意事项
