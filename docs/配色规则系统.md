## 🌈 第三层页面配色规则

### 配色组合公式
```
第三层背景色 = 地区色主调 + 活动色辅调
```

### 配色生成逻辑
1. **同色系组合**：地区色 = 活动色 → 使用色阶渐变
   - 例：东京祭典 = 红色(地区) + 红色(活动) = `from-red-100 to-red-200`

2. **不同色系组合**：地区色 ≠ 活动色 → 地区色主，活动色辅
   - 例：埼玉祭典 = 橙色(地区) + 红色(活动) = `from-orange-100 to-red-200`
   - 例：千叶祭典 = 天蓝(地区) + 红色(活动) = `from-sky-100 to-red-200`

### 具体配色示例

| 地区+活动 | 配色组合 | 生成结果 |
|-----------|----------|----------|
| 东京祭典 | 红+红 | `from-red-100 to-red-200` |
| 埼玉祭典 | 橙+红 | `from-orange-100 to-red-200` |
| 千叶祭典 | 蓝+红 | `from-sky-100 to-red-200` |
| 神奈川祭典 | 蓝+红 | `from-blue-100 to-red-200` |
| 北关东祭典 | 绿+红 | `from-green-100 to-red-200` |
| 甲信越祭典 | 紫+红 | `from-purple-100 to-red-200` |

### 使用方法

```typescript
// 方法1：自动配色（推荐）
<MatsuriPageTemplate
  region={regionConfig}
  events={events}
  regionKey="saitama"
  activityKey="matsuri"

/>
```

## 📋 AI操作规范

### 创建新页面时必须遵循：

1. **使用标准配色系统**
   - 传入正确的`regionKey`和`activityKey`
   - 移除手动`gradientColors`配置

2. **使用循环导航系统**
   - 传入正确的`regionKey`参数
   - 导航将自动生成，无需手动配置

3. **配色一致性检查**
   - 确保地区色符合首页设计
   - 确保活动色符合二层设计
   - 验证第三层配色符合组合规则

4. **技术验证**
   ```bash
   # 检查配色使用是否正确
   grep -r "gradientColors.*from-" src/app/*/matsuri/
   
   # 检查循环导航配置
   grep -r "regionKey.*activityKey" src/app/*/matsuri/
   ```
